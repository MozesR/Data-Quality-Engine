Help

Quick start, common workflows, and troubleshooting.

Quick Start

  1. Connect a MCP server in MCP Session.
  2. Login (top right shows your session).
  3. Run Assessment in Run Workflow.
  4. Review results in Workflow Output and export an Audit Pack.
  5. Adjust rules in DQ Rules Editor and re-run.

Glossary

  • Assessment: detects violations and computes quality index.
  • Correction: applies correction rules (e.g., fill defaults).
  • Suggestion: proposed new rule generated by LLM during assessment.
  • Audit Pack: CSV/PDF artifacts exported per run for compliance.

Troubleshooting

I get “Authentication required”

You are not logged in on one of the selected servers. Use the popup button Go to select MCP server, then login on that server.

Server not reachable / connection failed

Ensure Docker stack is running. Then go to MCP Session, connect to the server URL, and initialize it.

Rule editor shows red fields

Red inputs indicate required values are missing (Rule ID / Field). Fill them before saving.

Downloads do not work

Use Export Audit Pack inside a run detail. Then click Download for CSV/PDF artifacts.
